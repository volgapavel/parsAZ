

**Цель:** создать работающий прототип системы мониторинга СМИ, которая автоматически собирает новости с азербайджанских сайтов, извлекает из текстов информацию о компаниях и персонах и формирует структурированные данные для анализа репутационных и compliance-рисков.

**О компании.** parsAZ - технологический проект для автоматизации compliance-процессов. Основные пользователи - банки, инвестиционные фонды, финтех-компании и прочие компании в зарегулированных сферах.

**О продукте.** Платформа parsAZ помогает автоматизировать проверку контрагентов, мониторить репутационные риски и выявлять связи между компаниями и персонами. Система агрегирует данные из открытых источников, государственных реестров и СМИ.

**Проблема региона.** Азербайджанский рынок представляет особый интерес для финансовых институтов региона из-за развитой нефтегазовой отрасли и активной инвестиционной деятельности. При этом большинство значимой информации публикуется в местных СМИ на азербайджанском языке, что затрудняет мониторинг для международных команд.

**Т****екущая ситуация.** Сейчас сотрудники вручную просматривают новостные сайты, используют Google Translate и вручную составляют досье на компании. Это занимает 3–5 часов на одну проверку. Необходимо автоматизировать этот процесс.

#### Условия выполнения

**Целевая аудитория:**

- compliance-офицеры банков и финансовых компаний;
    
- risk-менеджеры инвестиционных фондов;
    
- аналитики по due diligence.
    

**Цели продукта:**

- Сократить время проверки одного контрагента с 3–5 часов до 15–20 минут.
    
- Автоматизировать сбор негативной информации о компаниях.
    
- Выявлять скрытые связи между персонами и организациями.
    

**Источники данных:**

- Новостные сайты: azərbaycan.az, Report.az, Trend.az, 1news.az, APA.az.
    
- Разделы: Бизнес, Экономика, Политика, Происшествия.
    
- Исторический период: последние 6–12 месяцев.
    
- Объем: не менее 200–300 статей.
    

**Ключевые сущности для извлечения:**

- Персоны: ФИО, должности.
    
- Компании и организации.
    
- События: назначения, отставки, суды, сделки, санкции.
    
- Даты событий, локации.
    

**Типы рисков:** коррупция и взяточничество, отмывание денег, мошенничество, санкции и штрафы, судебные разбирательства, банкротство, связи с криминалом, конфликт интересов.

**Ограничения:**

- Работа с азербайджанским языком (можно использовать промежуточный перевод на русский/английский).
    
- Обработка текстовых данных (видео/аудио не требуется).
    
- Фокус на структурированное извлечение данных, а не на сложный анализ тональности.
    

**В рамках решения кейса вам будет необходимо:**

1. Разработать парсер для 1–2 новостных сайтов.
    
2. Собрать корпус из 200–300 статей на азербайджанском языке.
    
3. Спроектировать и развернуть базу данных (PostgreSQL/MongoDB).
    
4. Изучить и протестировать NER-модели для азербайджанского языка или переведенного текста.
    
5. Настроить пайплайн: Текст → Предобработка → Извлечение сущностей → Сохранение в БД.
    
6. Разработать REST API с двумя эндпоинтами: обработка нового текста и поиск по базе новостей.
    
7. Упаковать решение в Docker и подготовить инструкцию по запуску.
    
8. Подготовить демо и презентацию с примерами работы системы.
    

**Информация по участию:**

- NDA не требуется.
    
- Кейс можно использовать в портфолио.
    
- Предусмотрены регулярные встречи с менторами для консультаций.
    

#### О заказчике задачи


**Зона ответственности ментора:** валидация финального решения.

**Формат взаимодействия:**

- Kick-off встреча в первую неделю.
    
- Asynchronous поддержка в Telegram-чате.
    

**Предоставляемые ресурсы:**

- Доступ к облачной инфраструктуре (опционально).
    
- API-ключи для инструментов (обсуждается отдельно).
    

#### Ожидаемый результат

Технологические требования:

- Язык программирования: Python 3.8+.
    
- База данных: PostgreSQL или MongoDB.
    
- Контейнеризация: Docker.
    
- REST API для взаимодействия с моделью.
    
- NLP библиотеки: spaCy (с multilingual моделями), transformers (HuggingFace), NLTK, stanza; NLP модели: mBERT (multilingual BERT), XLM-RoBERTa, любые open-source NER модели.
    

Необходимо предоставить заказчику:

- Парсер новостных сайтов (Python-скрипты для сбора данных с 2–3 источников. Сохранение в БД. Документация по запуску).
    
- База данных с минимум 200 новостями (Таблица статей: URL, дата, заголовок, текст, источник. Таблица сущностей: персоны и компании с типами. Связь статей и сущностей)
    
- NLP-модель для извлечения сущностей (Извлечение имен персон и компаний. Извлечение должностей. Может работать через перевод на русский/английский. Упакована в Docker-контейнер).
    
- API с документацией (Endpoint для обработки нового текста. Endpoint для поиска по базе. Swagger/OpenAPI документация).
    
- JSON-примеры обработанных статей (JSON-файл с 20–30 примерами обработанных статей. Структура: Исходный текст → Извлеченные сущности)
    
- Презентация (10–15 слайдов) с описанием подхода, архитектуры, метрик и примеров.
    

Формат сдачи:

- GitHub-репозиторий с кодом.
    
- Docker Compose для запуска всей системы.
    
- Dump базы данных с собранными новостями.
    

Метрики качества:

- Точность извлечения имен: >60%.
    
- Покрытие источников: 1–2 сайта.
    
- Объем базы: >200 статей.
    
- API response time: <10 сек. на статью.
